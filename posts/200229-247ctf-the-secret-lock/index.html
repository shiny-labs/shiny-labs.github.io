We need something efficient at solving constraints… Luckily, that’s one of the speciality of Satisfiability Modulo Theories (SMT) solvers. One of the widely used options for SMT solvers is the z3 Theorem Prover, an open source SMT solver from Microsoft Research.

Installation instructions for Archlinux:

# pacman -S z3 python-z3

To test that the setup is functional, run the following script:

#!/usr/bin/python

from z3 import *

x = Real('x')
y = Real('y')
s = Solver()
s.add(x + y > 5, x > 1, y > 1)

print(s.check())
print(s.model())

quit()

If everything works as intended, it should output:

sat
[y = 3/2, x = 4]

As it was my first time using z3, I made extensive use of this guide: Z3 API in Python which explains everything from the ground up and has lots of examples.

Let’s start our z3 Python script then!

The first thing we define is the format of the flag:

#!/usr/bin/python

from z3 import *

flag = [ BitVec("flag[{}]".format(i), 32) for i in range(40) ]

We use a list of 40 items, with each item being what z3 calls a Bit Vector of size 32. This allows us to represent each character in the flag.

Next, we define the constraints that the flag should satisfy:

// create a list of constraints
constraints = []
// parse the constraints from the JS code
with open("constraints.js", "r") as f:
    constraints = f.read().split("&&")

print("[-] Loaded {} constraints".format(len(constraints)))

// initiate a solver
s = Solver()

// add the constraints and specify they should be &&ed with each other.
for constraint in constraints:
    #print(constraint)
    s.add(And(eval(constraint)))

Since we are parsing the constraints and splitting on “&&”, the constraints list will contain elements like (flag[37] - flag[37]) * flag[15] == 0). Each item of the constraints list is added to the solver, specifying that the logical operation should be an AND.

Finally, we ask the solver if there’s a solution to this bunch of constraints. We make the hypothesis that the solution for each character is going to be unique, and that’s it’s going to be a int. Then, mash everything so it looks like a flag at the end.

if s.check() == sat:
    print("[-] yay, got a solution")
    m = s.model()
    l = []

    t = m.evaluate(flag[0])

    for i in range(40):
        v = m.evaluate(flag[i]).as_long()
        l.append(chr(v))
